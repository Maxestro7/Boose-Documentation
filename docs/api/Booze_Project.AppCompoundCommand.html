Class AppCompoundCommand

Base class for compound conditional commands (for example the application-level if/while blocks). Extends AppConditionalCommand and provides linking to the paired conditional (the corresponding if/end/else command) plus helper logic to normalise and evaluate conditional expressions.

internal class AppCompoundCommand : AppConditionalCommand, IAppCommand
Remarks

A compound command stores a reference to its paired AppConditionalCommand via CorrespondingCommand. The program/compiler must set this property when building the command list; callers should check for null before reading members on the corresponding command to avoid NullReferenceException at runtime. The class also validates that the compound command has exactly one parameter and that the parameter contains a comparison operator (<, > or =).

Remarks

Example: for an if block this should point to the corresponding end (or else) command so the control-flow logic can jump to the correct line. This property may be null until the program linking/compilation phase sets it.

Remarks
