Class AppEvaluation

Base class for variable evaluations and assignment-like commands. Provides storage for a variable name, an expression string and the evaluated result. Inherits common parameter handling from AppCommand.

Syntax
public abstract class AppEvaluation : AppCommand, IAppCommand
Remarks

Concrete evaluation commands (for example numeric, real and boolean variable types) derive from AppEvaluation to share parsing and evaluation behaviour. The compilation phase splits the textual parameter list into a variable name and an optional expression. At runtime Execute() evaluates the expression via the containing AppStoredProgram.

Compile()

Compile-time parsing for evaluation commands. Splits the raw parameterList on '=' to obtain the variable name and (optional) expression.

Declaration
public override void Compile()
Overrides
AppCommand.Compile()
Remarks

After calling base.Compile() this method:

  • Sets VarName to the left-hand side token.
  • If an expression is present, trims and stores it in Expression.
  • If no expression is present the Value is left as zero and Expression is empty.

Remarks

If Expression is an expression according to the containing program, it is evaluated via EvaluateExpression(string). The lower-cased, trimmed result is stored in evaluatedexpression for use by derived classes.
