Class AppParser

Converts textual program source lines into concrete IAppCommand instances and registers them with an AppStoredProgram.

Remarks

Responsibilities:

  • Normalises operator spacing to make tokenisation predictable.
  • Skips comment lines (lines beginning with *).
  • Splits a source line into a command name and parameter list and uses AppCommandFactory to create the command.
  • Assigns the created command to the target AppStoredProgram, invokes set(AppStoredProgram, string), then calls Compile(). The parser is intentionally lightweight â€” syntax adjustments (for example synthesising a declaration for simple assignments) are performed to make the factory-driven command creation simpler for common idioms.

Type Description

A compiled IAppCommand instance for the line, or null for lines that should be ignored (comments). The returned command has been assigned to the program via set(AppStoredProgram, string) and had Compile() invoked.
Remarks

Behaviour details:

  • Lines beginning with * are treated as comments and return null.
  • Operators are spaced to simplify token splitting.
  • The first token is interpreted as the command name; the remainder of the line becomes the parameter list.
  • For simple assignment syntax such as x = 1 + 2, if the first token is not a declaration (int or real), the parser synthesises an int declaration form so that the factory can create the appropriate evaluation command.
  • The factory is responsible for mapping the textual command name to a concrete command type; failure to find a mapping will result in a BOOSE.FactoryException from the factory.
