Class AppStoredProgram

In-memory representation of an executable application program. Holds an ordered list of AppCommand instances and provides execution, variable storage, expression evaluation and conditional-linking helpers used by the parser and command implementations.

Syntax
public class AppStoredProgram : ArrayList, IList, ICollection, IEnumerable, ICloneable, IAppStoredProgram
Remarks

Key responsibilities:

  • Inherits from ArrayList and stores program commands.
  • Maintains a drawing BOOSE.ICanvas instance which is assigned to canvas-aware commands when they are added.
  • Exposes a program counter via PC that indicates the next command index to execute.
  • Keeps an internal variable table (variablearr) containing AppEvaluation instances.
  • Uses an internal Stack (conditionals) to assist compilation/linking of compound conditional commands (for example if, else, end, while).
  • Call ResetProgram() to clear runtime and compile-time state before parsing/adding commands.

Run()

Executes the program from the beginning (sets PC to 0) until no commands remain.

Declaration
public void Run()
Remarks

Execution behavior:

  • Validates the program validity flag and throws BOOSE.StoredProgramException if the program is invalid.
  • Iterates while Commandsleft() is true:
    • Retrieves the current command and calls its Execute() implementation.
    • If the command did not modify PC, the method increments PC to advance to the next command.
    • If a BOOSE.BOOSEException is thrown by a command, its message(s) are collected and a BOOSE.StoredProgramException is thrown immediately after the command completes to surface runtime errors.
Exceptions
Type Condition
StoredProgramException

Thrown when the program is invalid at start or when runtime errors are encountered.
