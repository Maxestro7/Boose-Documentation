Namespace Booze_Project

Classes

AppBool

Represents a boolean evaluation value in the application. Inherits evaluation behavior from AppEvaluation and exposes a boolean view of the stored numeric value.

AppCanvas

Concrete application implementation of the BOOSE.ICanvas interface. Provides simple drawing primitives backed by an in-memory Bitmap and a Graphics surface. The canvas maintains a current pen position (Xpos, Ypos) used as the origin for many drawing operations.

AppCanvasCommand

Base class for application commands that operate on a drawing canvas. Inherits from AppCommand and exposes a protected BOOSE.ICanvas instance plus pen position fields used by concrete canvas commands (for example: rect, circle, moveto).

AppCircle

Represents the circle command. Validates a single integer parameter (radius) and instructs the canvas to draw a circle. Inherits from AppCommand1Param, the application-level single-parameter command base which derives from AppCanvasCommand.

AppCommand

Base application command implementing IAppCommand. Provides common parameter handling, a reference to the containing AppStoredProgram, and basic compile/execute lifecycle behaviour used by concrete command implementations.

AppCommand1Param

Base class for application commands that accept exactly one parameter. Derives from AppCanvasCommand and provides a standard parameter-count validation used by single-parameter canvas commands.

AppCommand2Param

Base class for application commands that accept exactly two parameters. Derives from AppCommand1Param and enforces a two-parameter count during compilation.

AppCommand3Param

Base class for application commands that accept exactly three parameters. Derives from AppCommand2Param and enforces a three-parameter count during compilation.

AppCommandFactory

Singleton factory that creates application-specific IAppCommand instances. Maps textual command names to concrete App* implementations used by the application.

AppCompoundCommand

Base class for compound conditional commands (for example the application-level if/while blocks). Extends AppConditionalCommand and provides linking to the paired conditional (the corresponding if/end/else command) plus helper logic to normalise and evaluate conditional expressions.

AppConditionalCommand

Base class for commands that represent conditional behaviour (blocks/expressions). Inherits boolean evaluation behavior from AppBool and adds line-number bookkeeping and a runtime condition used to control program flow.

AppElse

Represents an application-level else compound command. Acts as the partner for a preceding conditional (if) and links to the subsequent end command.

AppEvaluation

Base class for variable evaluations and assignment-like commands. Provides storage for a variable name, an expression string and the evaluated result. Inherits common parameter handling from AppCommand.

AppInt

Application implementation for integer variable evaluation/assignment. Inherits parsing and evaluation behaviour from AppEvaluation and updates the containing program's integer variable store on successful evaluation.

AppMoveTo

Application implementation of the moveto command. Reads two integer parameters (x, y) and instructs the configured canvas to move the pen to that location.

AppParser

Converts textual program source lines into concrete IAppCommand instances and registers them with an AppStoredProgram.

AppPenColor

Application implementation of a pen colour command. Parses three integer parameters (red, green, blue) and applies them to the active canvas pen.

AppReal

Application implementation for real (double) variable evaluation/assignment. Inherits parsing and evaluation behaviour from AppEvaluation and stores the evaluated value as a double.

AppRect

Application implementation of the rect command. Validates two integer parameters (width and height) and instructs the configured canvas to draw a rectangle.

AppStoredProgram

In-memory representation of an executable application program. Holds an ordered list of AppCommand instances and provides execution, variable storage, expression evaluation and conditional-linking helpers used by the parser and command implementations.

AppWrite

Implements the application-level write command. Resolves tokens in the parameter list (variables or literals), evaluates the assembled expression, and writes the resulting text to the configured canvas via Canvas.WriteText.

Append

Represents the application-level end command that terminates a compound block.

Appif

Application implementation of the if compound command. Evaluates the conditional expression and, if the condition is false, advances the program counter to the command indexed by EndLineNum.

ApplicationConfiguration

Bootstrap the application configuration.

Appwhile

Application implementation of the while compound command. Evaluates a boolean expression each iteration and, when false, advances the program counter to the linked end command.

Booze_UI

Main Windows Forms UI for the BOOSE application. Hosts the drawing canvas, command factory, parser and stored program, and exposes event handlers for user actions.

Program

Interfaces

IAppCommand

Defines the contract for application command objects. Implementations represent executable program statements that can be compiled, validated and executed within an AppStoredProgram.

IAppCommandFactory

Defines a factory for creating application command instances. Implementations map textual command type names (as produced by the parser) to concrete IAppCommand objects.

IAppParser

Defines the contract for a parser that converts textual program source into application command instances.

IAppStoredProgram

Defines the runtime program container used by the application. Provides storage for commands, variables, conditional linking and expression evaluation.
